## Quick Start

If you're new to the codebase, we recommend reading the documentation in this order:

1. Start with **Architecture Overview** to understand the big picture
2. Read **Directory Structure** to learn where everything is
3. Dive into **Component Guide** to understand individual pieces
4. Explore **Canvas System** if you're working on drawing features
5. Check **Enhancement Roadmap** for contribution opportunities

## Getting Help

If you encounter issues:

1. Check the **Troubleshooting Guide** first
2. Review the **Development Guide** for setup problems
3. Look at the specific component documentation for feature-specific issues

## Contributing
